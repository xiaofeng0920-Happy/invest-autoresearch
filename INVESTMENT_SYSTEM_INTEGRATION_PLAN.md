## 🎯 目标

将所有投资相关技能整合到一个统一的工作流中，通过 `investment-leader` 调度中心自动协调各技能协作，实现从数据收集到报告生成的全流程自动化。

## 📋 完整技能清单与分工

### 1️⃣ 数据层 - 收集所有输入数据

| 技能 | 职责 | 数据源 |
|------|------|--------|
| **data-collector** | 股价、财报、指数、新闻 | akshare、tushare、富途、东方财富 |
| **eastmoney-financial-search** | 金融新闻/公告/研报搜索 | 东方财富妙想 |
| **ocr-statement** | 识别股票结单/对账单 | 本地图片/PDF |

---

### 2️⃣ 分析层 - 处理和分析数据

| 技能 | 职责 | 输出 |
|------|------|------|
| **portfolio-analyzer** | 计算盈亏、对比成本 | 盈亏表、盈亏比例 |
| **transaction-classifier** | 交易分类（买入/卖出/分红） | 分类报告、税务数据 |
| **hk-stock-predictor** | 港股预测分析 | 港股预测报告 |

---

### 3️⃣ 策略层 - 生成投资建议

| 技能 | 职责 | 风格 |
|------|------|------|
| **strategy-advisor** | 综合策略建议 | 买入/卖出/持有 |
| **buffett-investor** | 价值投资分析 | 巴菲特风格 |
| **value-investor** | 价值投资分析 | 格雷厄姆风格 |
| **growth-investor** | 成长投资分析 | 费雪 + 林奇风格 |
| **contrarian-investor** | 逆向投资分析 | 困境反转 |
| **risk-officer** | 风险评估 | 风险评分、一票否决 |
| **debater** | 质疑和替代方案 | 风险提示、备选方案 |

---

### 4️⃣ 执行层 - 交易和回测

| 技能 | 职责 | 功能 |
|------|------|------|
| **futu-trading-bot** | 富途交易执行 | 港股/美股实盘 |
| **paper-trading** | 模拟交易 | 虚拟组合跟踪 |
| **backtester** | 策略回测 | 历史数据验证 |
| **港股量化投资** | 量化选股 + 组合优化 | 因子模型、权重优化 |

---

### 5️⃣ 输出层 - 报告和推送

| 技能 | 职责 | 输出格式 |
|------|------|----------|
| **report-writer** | 撰写投资报告 | Markdown |
| **message-dispatcher** | 推送报告 | 飞书/微信/邮件 |

---

### 6️⃣ 调度中心 - 统一协调

| 技能 | 职责 |
|------|------|
| **investment-leader** | 任务拆解、派单、汇总、异常处理 |

## 🔄 完整工作流程

### 流程 1：每日持仓报告（定时任务）

```
09:00 → investment-leader 触发
   ↓
   → data-collector 收集实时股价、财报、指数
   ↓
   → portfolio-analyzer 计算盈亏
   ↓
   → strategy-advisor 生成调仓建议
   ↓
   → risk-officer 风险评估
   ↓
   → report-writer 撰写报告
   ↓
   → message-dispatcher 推送到飞书
```

---

### 流程 2：深度投资分析（手动触发）

```
用户："分析一下腾讯要不要加仓"
   ↓
investment-leader 拆解任务：
   ↓
   → data-collector 收集腾讯数据（股价、财报、新闻）
   → hk-stock-predictor 港股预测分析
   → buffett-investor 价值投资分析
   → growth-investor 成长投资分析
   → contrarian-investor 逆向投资分析
   → debater 提出质疑和替代方案
   → risk-officer 风险评估
   ↓
   → strategy-advisor 综合所有意见生成建议
   ↓
   → report-writer 撰写深度分析报告
```

---

### 流程 3：财报季特别分析

```
触发：持仓股票发布财报
   ↓
   → eastmoney-financial-search 搜索财报新闻/分析师评论
   → data-collector 获取最新财报数据
   → portfolio-analyzer 对比持仓成本
   → value-investor 评估估值变化
   → growth-investor 评估成长性变化
   ↓
   → strategy-advisor 生成调仓建议（加仓/减仓/持有）
```

---

### 流程 4：调仓 rebalance（定期执行）

```
触发：每月/每季度第一天
   ↓
   → portfolio-analyzer 计算当前持仓权重
   → 港股量化投资 组合优化（目标权重）
   → strategy-advisor 对比当前 vs 目标
   → risk-officer 风险评估
   ↓
   → 生成调仓清单（买入 X 股 Y，卖出 Z 股 W）
   → paper-trading 模拟执行
   → 用户确认后 → futu-trading-bot 实盘执行
```

## 🚀 使用方式

### 方式 1：语音/文字指令（最简单）

直接说：
- "生成今日持仓报告"
- "分析一下腾讯要不要加仓"
- "这个月要不要调仓"
- "看看财报季有什么需要关注的"

→ `investment-leader` 自动拆解任务并调度所有相关技能

---

### 方式 2：定时任务（自动化）

在 `HEARTBEAT.md` 或 cron 中配置：
```
每天 09:00 → 早间报告
每天 21:00 → 晚间总结
每月 1 日 → 月度调仓分析
财报季 → 财报特别分析
```

---

### 方式 3：手动触发完整流程

```python
# 触发每日报告任务
sessions_send(
    sessionKey="agent:investment-leader:main",
    message='{"task": "daily_report", "holdings_file": "memory/锋哥持仓.md"}'
)
```

## ⚙️ 配置说明

### 持仓文件格式

```markdown
# 锋哥持仓

## 美股
- GOOGL: 10 股，成本 $250
- AAPL: 20 股，成本 $220
- ...

## 港股
- 00700 (腾讯): 100 股，成本 HKD 450
- 09988 (阿里): 200 股，成本 HKD 100
- ...
```

### 风险偏好配置

在 `MEMORY.md` 或单独配置文件：
```yaml
risk_tolerance: medium
max_single_position: 20%  # 单只股票最大仓位
max_sector_exposure: 40%  # 单行业最大暴露
rebalance_frequency: monthly  # 调仓频率
```

## 📊 输出示例

### 每日报告包含：
1. **持仓总览** - 总市值、当日盈亏、总盈亏
2. **个股表现** - 每只股票价格变化、盈亏比例
3. **市场动态** - 大盘指数、重要新闻
4. **财报提醒** - 近期发布财报的股票
5. **策略建议** - 调仓建议、风险提示

### 深度分析报告包含：
1. **公司基本面** - 财务数据、业务分析
2. **估值分析** - PE、PB、PEG、DCF
3. **技术面** - 趋势、支撑/阻力位
4. **多方观点** - 价值/成长/逆向分析
5. **风险提示** - 风险官评估、辩论员质疑
6. **操作建议** - 买入/卖出/持有 + 目标价

## ⚠️ 注意事项

### 1. 数据源优先级
- 港股 → 富途 OpenD（实时）
- A 股 → akshare/tushare
- 美股 → akshare（有延迟）或富途

### 2. API 限流
- 批量查询时注意频率限制
- 使用缓存减少重复请求

### 3. 任务超时
- 单个 Agent 超时 5-10 分钟
- 完整流程超时 30 分钟
- 超时后自动重试或降级

### 4. 实盘交易
- 默认使用模拟交易（paper-trading）
- 实盘需要用户二次确认
- 风控官有一票否决权
